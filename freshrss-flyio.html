Hosting FreshRSS on fly.io

2025-11-09

The problem.

I'm broke, and I like using FreshRSS. Here's how to self-host it on fly.io for free.

The solution.

I assume you already have the account on fly.io.

!NOTE In this guide will be used my modified container with extensions I prefer to have. If you want to use official container feel free to swap ghci.io/olexsmir/freshrss-image to docker.io/freshrss/freshrss. Also feel free to fork and modify github.com/olexsmir/freshrss-image according to your needs.

  1. Create an app on fly.io,
fly app create your_app_name
  1. Create fly.toml file with following contents:
app = 'your_app_name' # TODO: update me
primary_region = 'otp'
kill_signal = 'SIGINT'
kill_timeout = '5s'

[experimental]
  auto_rollback = true

[build]
  image = 'ghcr.io/olexsmir/freshrss-image:1.27.1'

[env]
  CRON_MIN = '*/20'

[[mounts]]
  source = 'freshrss_data'
  destination = '/var/www/FreshRSS/data'

[[services]]
  protocol = 'tcp'
  internal_port = 80
  processes = ['app']

  [[services.ports]]
    port = 80
    handlers = ['http']
    force_https = true

  [[services.ports]]
    port = 443
    handlers = ['tls', 'http']

  [services.concurrency]
    type = 'connections'
    hard_limit = 5
    soft_limit = 3

  [[services.tcp_checks]]
    interval = '15s'
    grace_period = '1s'

[[vm]]
  memory = '512mb'
  cpu_kind = 'shared'
  cpus = 1
